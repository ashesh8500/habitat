1. Trajectory Collection

We load the HM3D minival scene 00800-TEEsavR23oF — a 398 m² furnished residential space across 2 floors and 7 rooms — into habitat-sim 0.3.3. An agent with a pinhole RGB camera at 1.5 m eye height (90° HFOV, 640×480) navigates via navmesh-constrained waypoints. The pathfinder's random navigable points are perturbed with directional walk heuristics to produce natural exploration patterns spanning 16.6 m × 7.7 m of walkable area. 26 frames are captured with per-frame position and quaternion rotation metadata.

2. Object Detection

YOLO-World v2 (yolov8s-worldv2.pt, 24.7 MB) operates in open-vocabulary mode — objects are specified by natural language prompts mapped through a CLIP text encoder, requiring no retraining for new categories. Our vocabulary spans 41 indoor classes at confidence threshold 0.15.

3. Depth Estimation & Segmentation

Depth Pro (Apple, 2024) estimates metric depth from a single RGB frame — no stereo pair or LiDAR required. The 1.8 GB ViT-based model outputs pixel-level depth in meters. FastSAM provides lightweight segmentation from YOLO-World bounding box prompts; masks are resized from FastSAM's internal 640 px resolution to the original frame dimensions via nearest-neighbor interpolation.

4. Setup & Reproducibility

The pipeline uses two isolated Python environments to resolve dependency conflicts between habitat-sim's native C++ bindings (requiring numpy < 2, Magnum, Pillow 10.4) and the ML stack (PyTorch 2.11, numpy 1.26). A single script orchestrates all three stages:

bash run_pipeline.sh

Individual stages can be run independently for debugging or distributed execution:

# Stage 1 — Trajectory (conda: habitat-sim)
conda run -n habitat-sim python scripts/collect_trajectory.py \
  --scene data/hm3d/minival/00800-TEEsavR23oF/TEEsavR23oF.basis.glb \
  --frames 25

# Stage 2 — Detection (uv venv: ML models)
source .venv/bin/activate
python scripts/detect_objects.py --confidence 0.15

# Stage 3 — Depth + Segmentation + Analysis
python scripts/analyze_scene.py --max-frames 5

6. Design Decisions

Modular stages. Each stage communicates via JPEG and JSON — standard formats that enable independent debugging, model swapping, and distributed execution. Replacing YOLO-World with DETR requires no changes to other stages.

Open-vocabulary detection. The vocabulary is a CLI argument — changing from “chair, table” to “fire extinguisher, exit sign” requires no retraining, only a text prompt change.

Navmesh-constrained navigation. Random camera positions produce unnatural trajectories. The navmesh restricts waypoints to walkable regions, producing paths that resemble human movement through the 398 m² space.

FastSAM as NanoSAM proxy. NanoSAM targets NVIDIA Jetson edge devices with TensorRT. FastSAM provides architecturally equivalent functionality (bounding box → mask) through the ultralytics ecosystem with no custom build required.
