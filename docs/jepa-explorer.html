Can JEPA replace the spatial reasoning stack?

A visual study page for exploring whether joint-embedding predictive models can substitute for YOLO-World, Depth Pro, and FastSAM in the Habitat/HM3D interview pipeline.

This is not a new implementation claim. It is a structured experiment design: what exists, what is missing, and what would be worth testing next.

No direct swapJEPA weights output embeddings, not boxes, masks, or metric depth.
Good probe targetFrozen I-JEPA features can be tested for depth structure with a small decoder.
Best available weightsfacebook/ijepa_vith16_1k and JEPA-WM DROID.
Viable extensionUseful as a research experiment, not as a baseline replacement.
Practical conclusion. Keep the current pipeline as the verified baseline. Add a JEPA branch that learns probes from frozen representations and compare against Depth Pro / YOLO-World outputs. That gives a credible, controlled experiment without weakening the task deliverable.

1. The baseline produces explicit scene measurements

The existing repository already solves the interview task using specialist models. Each model returns an artifact that is immediately measurable: a bounding box, a mask, or a metric depth value. JEPA models are different: their native output is a latent representation.

YOLO-World is the open-vocabulary detector: it turns a prompt vocabulary into boxes, labels, and confidences. A bare JEPA encoder does not provide this interface.

2. JEPA is useful because the representation is spatial, not because it is already a detector

In I-JEPA, the model receives visible context patches and predicts the representation of hidden target patches. The prediction happens in embedding space. This tends to preserve semantic layout while discarding pixel-level texture. For this project, the useful question is: does that representation contain enough information to recover depth and object layout?

fθ(image) → z ∈ ℝ28×28×1280;   gφ(z) → depth or objectness.   Train φ, keep θ frozen.

    5. A controlled experiment that would be worth implementing

    The cleanest next step is to avoid overclaiming. Use the verified pipeline to generate pseudo-labels, then measure whether frozen JEPA features can recover those labels with small trainable heads.

    6. Decision guide

    Do now

    Build scripts/jepa_depth_probe.py. Freeze I-JEPA, train a small decoder on HM3D frames with Depth Pro pseudo-labels, and compare per-object depth statistics.

    Do next

    Try JEPA-WM DROID embeddings on consecutive Habitat frames. Test whether latent distance tracks camera translation and rotation from the simulator metadata.

    Do not claim

    Do not say JEPA replaces the full perception stack until a trained head returns boxes, masks, labels, and depth at comparable quality.

    Bottom line. JEPA is viable as an experimental branch for representation-based spatial reasoning. It is not currently a drop-in substitute for the specialist baseline. That distinction is the strongest way to present the idea.
