JEPA as a spatial representation substitute

Assessing whether joint-embedding predictive architectures can replace specialist models in the HM3D perception pipeline.

← Habitat pipeline · NYU WIRELESS · Summer 2026

1. Baseline overview

The existing pipeline uses three specialist models, each producing an explicit, directly measurable output: YOLO-World returns bounding boxes with class labels, Depth Pro returns a metric depth map in meters, and FastSAM returns per-object instance masks. These artifacts feed into a spatial analysis stage that computes per-object depth statistics, producing structured JSON and side-by-side visualizations.

2. What JEPA provides

Joint-embedding predictive architectures (I-JEPA, V-JEPA, JEPA-WM) learn representations by predicting hidden patch embeddings from visible context. Their native output is a latent embedding field — not bounding boxes, not class labels, not metric depth, and not instance masks. The key question is whether these representations contain sufficient spatial structure to recover the same measurements through learned readout heads.

In I-JEPA (Assran et al., 2023), the context encoder processes visible image patches and a predictor estimates the representation of masked target patches. The loss operates in embedding space, which tends to preserve semantic layout while discarding pixel-level texture. For this project, the relevant question is: does that representation contain enough information to recover depth ordering and object layout?

No JEPA-family model provides object labels, boxes, masks, or metric depth out of the box. The most plausible substitution target is depth estimation, since Depth Pro pseudo-labels provide dense supervision and the spatial structure of the embedding field may support metric scale recovery with a lightweight decoder.

4. Candidate models

I-JEPA ViT-H/16

The best first candidate. It is a static image encoder with a HuggingFace checkpoint (facebook/ijepa_vith16_1k) at 448 × 448 native resolution, producing a 28 × 28 × 1280 embedding field. At ~0.6B parameters, it is manageable for single-GPU probing experiments. A depth probe can be trained on HM3D frames supervised by Depth Pro outputs, with evaluation on per-object depth statistics.

V-JEPA / V-JEPA 2

Video JEPA models encode spatiotemporal structure and may be useful for temporal consistency across trajectory frames. However, they are heavier to integrate and less directly applicable to the still-frame analysis required by the task. Checkpoints are available via facebookresearch/jepa.

JEPA-WM DROID

Conceptually the most aligned with egocentric spatial dynamics: it predicts future latent states from observations and actions. The DROID variant (facebook/jepa-wms, loaded via torch.hub) uses a DINOv3 ViT-L/16 encoder. Its action space is robotics-oriented, but the pose-conditioned prediction objective is a natural fit for trajectory data from Habitat-sim.

LeJEPA / Core-JEPA

Newer representation work, but the available public checkpoint (gajeshladhar/core-jepa) is trained on satellite imagery and has not been evaluated on indoor scenes. Domain mismatch makes this a lower-priority candidate.

6. Findings

JEPA-family models do not currently provide drop-in substitutes for the specialist detection, depth estimation, and segmentation stack. The native output is a latent representation, not the structured artifacts (boxes, labels, masks, metric depth) required for the spatial reasoning task. The baseline pipeline remains the verified solution for the task requirements.

However, a controlled experiment — training small readout heads on frozen I-JEPA features supervised by Depth Pro pseudo-labels — could assess whether the learned representations contain recoverable depth structure. This would serve as a principled research extension demonstrating judgment about representation learning, rather than a claim of replacement. The experimental design above outlines a self-contained probe that can be implemented in a single additional script without modifying the existing pipeline.
